SMPTE Standard Suite

SMPTE ST 2110 System

The professional framework for transporting media over managed IP networks — video, audio, and ancillary data as separate, synchronized elementary streams in real-time broadcast production.

🏆 2025 Emmy® Award — Outstanding Achievement in Engineering, Science & Technology
💡 Core System Philosophy
🔀

Essence Separation

ST 2110 breaks apart video, audio, and ancillary data into independent IP streams (essences), rather than bundling them together as SDI does. Each essence travels its own path across the network.

⏱️

PTP Precision Timing

Uses IEEE 1588 Precision Time Protocol (PTP) to synchronize all streams to sub-microsecond accuracy, ensuring perfect alignment at the receiver regardless of which network path each packet took.

🌐

IP Network Flexibility

Runs on standard managed Ethernet with Unicast and Multicast support, enabling software-defined routing of thousands of streams across facilities, campuses, and cloud environments.

10
ST 2110-10 System Timing & Definitions

The architectural foundation of the entire system. Defines the timing model based on PTP (IEEE 1588), links RTP timestamps to a shared reference clock, and establishes common requirements applicable to all essence streams. Without this, no synchronization across essences is possible.

PTP IEEE 1588 RTP Timestamps SDP UDP/IP
20
ST 2110-20 Uncompressed Video Transport

Specifies RTP/UDP transport of uncompressed video essence. Covers only the active picture area (excluding blanking intervals), saving 15–30% bandwidth compared to SDI. Supports resolutions up to 32K×32K pixels, full HDR, and flexible color modes. Built on IETF RFC 4175.

4K / 8K / HDR RFC 4175 ~24 Gbps (4K60) YCbCr / RGB
21
ST 2110-21 Video Traffic Shaping

Addresses the IP burstiness problem for video. Defines three sender classes: Narrow Linear (NL) at constant bit rate, Narrow (N) pausing during VBI, and Wide (W) for flexible software implementations. A type-W receiver must accept streams from any sender class.

NL Sender N Sender W Sender Traffic Shaping
22
ST 2110-22 Compressed Video Transport

Enables compressed video transport without mandating a specific codec, as long as it provides constant bitrate and low latency. The dominant implementation uses JPEG XS — a visually lossless, lightweight codec delivering under 1 ms latency, ideal for live production and cloud/remote workflows.

JPEG XS <1ms Latency Cloud / REMI Constant Bitrate
30
ST 2110-30 PCM Audio Transport

Defines uncompressed PCM 24-bit audio transport over RTP, based on the AES67 profile. Enables seamless interoperability with existing audio-over-IP ecosystems including Dante and RAVENNA. Supports up to 8 channels per stream at approximately 1.5 Mbps. Audio must also lock to PTP to prevent drift.

PCM 24-bit AES67 48 kHz ~1.5 Mbps/8ch
31
ST 2110-31 AES3 Audio Transport

Extends Part 30 to carry non-PCM audio signals encapsulated as AES3 data payloads using the ST 337 format. Used for passing compressed audio formats such as Dolby-E, Dolby AC3, and other broadcast-grade surround audio codecs without decoding and re-encoding.

AES3 / ST 337 Dolby-E Non-PCM Transparent
40
ST 2110-40 Ancillary Data (ANC)

Provides a generic way to encapsulate ST 291-compliant ancillary data over IP, completely independent of video and audio streams. Carries closed captions, subtitles, VANC/HANC data, copy protection signals, slate information, vertical interval test signals, and timecode metadata.

Closed Captions Timecode Tally / Slate ST 291
59
ST 2059 Broadcast PTP Timing Profile

The broadcast-specific PTP timing standard derived from IEEE 1588. Defines the SMPTE Epoch and the Grandmaster Clock hierarchy. A GPS/GNSS-disciplined Grandmaster distributes timing to all devices on the network with accuracy below one microsecond, enabling lock-step synchronization of all essence streams.

Grandmaster Clock GPS / GNSS <1 μs Accuracy IEEE 1588
22
ST 2022-7 Seamless Stream Protection

Provides hitless, seamless redundancy by sending two identical copies of every stream over two completely separate network paths. The receiver selects the best packets in real time, achieving zero packet loss even when one entire path fails — far superior to traditional failover with interruption.

Dual Path Zero Loss Hitless Failover Redundancy

⏱️ Why is PTP Essential?

Because ST 2110 sends video, audio, and data as completely separate IP streams that may travel different paths through the network, a common high-precision clock is essential to reassemble them correctly at the destination.


PTP gives every device on the network a shared clock accurate to less than one microsecond, ensuring a video frame and its associated audio arrive perfectly aligned — preventing lip-sync errors and enabling clean live switching.


Profile used: SMPTE ST 2059-2 — a broadcast-specific PTP profile built on IEEE 1588, mandated by the EBU for professional IP facilities.

🎛️ NMOS Specifications — Management & Control Layer
IS-04

Discovery & Registration

Enables ST 2110 devices to automatically announce themselves on the IP network and register their capabilities in a central Registry server. Every device publishes the senders and receivers it exposes, making them instantly discoverable by a broadcast controller.

IS-05

Connection Management

Provides a standardized REST API for a broadcast controller to establish, modify, or tear down connections between senders and receivers — in any vendor combination. Replaces proprietary control APIs with a single interoperable mechanism for IP routing.

IS-07

Events & Tally

The IP equivalent of GPIO signals in SDI environments. Sends Tally indicators, on-air/preview events, and real-time triggers across the network rather than physical wiring, enabling scalable tally systems and control event distribution.

IS-08

Audio Channel Mapping

Enables flexible routing of individual audio channels from multiple source streams into a single destination. Allows a broadcast controller to remap and reorder audio channels without reconfiguring the physical audio infrastructure.

IS-09

System Parameters

Enables automatic distribution of system-wide parameters such as PTP server addresses, DNS configuration, and authorization endpoints. Significantly reduces manual configuration overhead in large-scale ST 2110 deployments.

IS-11

Sender / Receiver Capabilities

Allows devices to advertise their supported AV formats to the broadcast controller, ensuring format compatibility is checked before a connection is established. Prevents mismatches in resolution, frame rate, or color format at connection time.

⚖️ Comparison: Legacy SDI vs. SMPTE ST 2110

🔌 Legacy SDI

  • ✗ Single multiplexed stream (video + audio + data tightly coupled)
  • ✗ Routing requires large, expensive dedicated SDI routers
  • ✗ Limited cable distance (≈300 m for HD over coax)
  • ✗ Format changes require new hardware at every point
  • ✗ Rigid point-to-point signal architecture
  • ✗ One BNC cable per signal — inflexible wiring

🌐 SMPTE ST 2110

  • ✓ Separate independent essence streams — maximum flexibility
  • ✓ Software-defined routing over standard IP network infrastructure
  • ✓ Unlimited distances across network — campus, city, cloud
  • ✓ Format-agnostic — supports 4K, 8K, HDR without hardware changes
  • ✓ Cloud-native, REMI, and remote production workflows
  • ✓ One fiber carries thousands of simultaneous streams
